| 论文的主要内容与要求 | **一、内容**<br>随着房地产交易与中介/渠道销售业务的信息化需求提升，房源、客户、带看、交易、回款、佣金与团队协作等数据在传统人工管理模式下容易出现“信息分散、流程缺失、难以追溯、口径不统一”等问题。本课题要求设计并实现一套“房产销售管理系统（RESMS）”，采用前后端分离架构实现房产销售业务的核心闭环管理，并提供基础经营统计分析能力。<br><br>（1）系统总体目标<br>实现“房源管理 → 客户管理 → 带看记录 → 交易管理（含状态流转） → 回款管理 → 佣金管理 → 团队与权限控制 → 统计报表”的完整业务闭环，系统需可部署、可演示、可复现。<br><br>（2）系统主要模块与功能要求<br>1) 用户与权限管理：实现登录鉴权（建议JWT），用户管理、角色管理、权限控制（至少包含管理员/销售经理/销售顾问/财务四类角色），并实现数据权限（销售仅可见本人数据；经理/管理员可见全局或团队数据；财务仅处理财务相关）。<br>2) 房源管理：房源信息的新增、编辑、删除、详情、分页查询与筛选（面积/价格/户型/状态/负责人等）；支持房源图片管理与封面图；支持房源审核与状态管理（待审核/在售/已预订/已成交/下架）。<br>3) 客户管理：客户信息的新增、编辑、详情、分页查询与筛选；客户归属/分配；包含意向信息（预算、区域、意向等级等）。<br>4) 带看管理：创建带看记录（关联客户与房源），支持查询与统计；带看结果与反馈可记录。<br>5) 交易管理：创建交易（关联房源、客户、销售人员）；实现交易状态流转（至少：待付定金→已付定金→已付首付→已过户→已完成/可取消），并实现关键业务规则（房源必须在售才可创建交易；同一房源不得存在多笔进行中交易；交易状态变化需联动房源状态）。<br>6) 回款管理：登记收/付款记录（金额、时间、方式、凭证/备注、关联交易），并进行权限控制；必要时实现“确认”流程。<br>7) 佣金管理：实现佣金生成与查询（至少固定比例或可配置比例），佣金需关联交易与销售人员；可增加审核/确认。<br>8) 统计与报表：至少实现2个统计页面（如成交额/成交量按时间统计、销售业绩排行、回款统计等），支持导出。<br>9) 日志与通知：关键业务操作记录操作日志；支持站内通知/消息推送（可选WebSocket）。<br><br>（3）数据库设计要求<br>完成ER图与表结构设计，至少包含用户/角色/权限、房源/图片/项目（新房/二手房扩展）、客户、带看、交易、回款、佣金、通知/日志等核心表；提供建表与初始化数据脚本（包含不同角色默认账号及演示数据）。<br><br>（4）系统工程与论文要求<br>系统需满足分层清晰、接口规范、统一异常处理、关键业务事务一致性；提供README运行说明、接口文档、测试用例与测试报告。完成不少于8000字毕业论文（或设计说明书），内容需包含需求分析、总体设计、数据库设计、关键技术与实现、测试、总结与展望，并体现至少15篇参考文献。 |  |  |  |  |  |  |
