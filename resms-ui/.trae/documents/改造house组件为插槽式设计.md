# 改造house组件为插槽式设计

## 改造目标
将house目录下的组件改为使用插槽(slot)的方式，提高组件的复用性和灵活性，允许父组件自定义组件内容。

## 改造方案

### 1. HouseCard组件改造
- 将卡片内部各部分改为插槽：
  - `header` 插槽：卡片头部（标题、价格）
  - `address` 插槽：地址信息
  - `info` 插槽：户型、开盘时间等信息
  - `features` 插槽：特色标签
  - `actions` 插槽：操作按钮
- 保留基本卡片结构，移除硬编码内容

### 2. HouseList组件改造
- 将房源列表部分改为插槽：
  - `header` 插槽：列表头部（标题、计数、操作）
  - `empty` 插槽：空状态
  - `default` 插槽：房源列表内容（用于注入HouseCard）
  - 保留分页功能
- 移除内部对HouseCard的直接依赖

### 3. FilterArea组件改造
- 将筛选条件部分改为插槽：
  - `filter-items` 插槽：筛选条件项
  - `selected-filters` 插槽：已选条件
- 保留筛选容器结构
- 移除内部对Pinia store的直接依赖

### 4. 调整Project.vue页面
- 更新组件使用方式，使用插槽注入内容
- 保持原有功能不变

## 改造步骤
1. 改造HouseCard组件为插槽式设计
2. 改造HouseList组件为插槽式设计
3. 改造FilterArea组件为插槽式设计
4. 更新Project.vue页面，使用新的插槽式组件
5. 测试功能是否正常

## 预期效果
- 组件复用性提高，可在不同场景下使用
- 组件灵活性增强，支持自定义内容和布局
- 保持原有功能不变
- 代码结构更清晰，职责更单一
